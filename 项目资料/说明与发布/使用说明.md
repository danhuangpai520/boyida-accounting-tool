# 保谊达车队做账工具使用说明

## 1. 最简单分享方式

只发这个文件：

`做账执行工具.exe`

对方双击即可打开。默认使用在线 GLM-OCR，不要求对方安装 Python、PaddleOCR、CUDA 或 5090 驱动环境。

在线 OCR 需要：

- 能联网；
- 默认内置 Key 可用，必要时也可以手动改成自己的 ZHIPU API Key；
- 图片单张不超过 GLM-OCR 官方限制。

这个版本支持 EXE 内置默认 API Key，也支持使用者在界面里手动修改。优先级是：环境变量 `ZHIPU_API_KEY` > 本机保存的 Key > EXE 内置默认 Key。

如果要重新生成“内置默认 Key”的 EXE，把 Key 放到 `builtin_api_key.txt`，然后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\项目资料\开发源码\设置内置Key并打包.ps1
```

打包完成后删除 `builtin_api_key.txt`。

每次打包都会自动在 `项目资料\版本备份` 里保存一份本地回退版本。新版备份包含 exe、源码、执行规范、使用说明和 SHA256 校验信息；备份不会保存明文 API Key。

## 2. 最终输出

软件固定生成：

`运输录入_<日期范围>.xlsx`

Excel 固定包含：

- `运输录入`
- `重复图片`

主表默认 9 列：

`日期、车号、驾驶员、重量、货物名称、装货地、收货地、备注、原图`

点击“一键出表”后，软件会先弹出表格列确认窗口：

- 直接点“默认生成”：按上面 9 列生成。
- 需要特殊批次多一列、改列名、调顺序、改列宽时，先在窗口里调整，再点“保存并生成”；新增列、删除列和高级字段映射在表格上右键操作。
- 自定义列模板会保存为 `excel_template.json`，下次打开软件仍可继续用。
- 原图方式默认是“外部原图链接”：Excel 只放可点击文件名，原图复制到同名 `_原图` 文件夹，表格更轻、更好传。
- 最终必须保留 `原图` 列，否则不能追溯对应原图。

## 3. 使用步骤

1. 双击 `做账执行工具.exe`。
2. 软件会自动创建当天批次总目录，例如 `2026-06-02_保谊达做账表`，把本批单据图片放进里面的 `待处理图片` 文件夹，或点“导入图片”选择本批图片。
3. OCR 引擎默认保持“在线智谱 GLM-OCR（默认，发同事推荐）”。
4. `ZHIPU API Key` 默认已填；要换 Key 时直接在框里改，保持勾选“记住 Key（仅本机）”。
5. 需要查看 OCR 推荐、检查环境、单独 OCR 或安装本地 OCR 时，点“OCR 设置”。
6. 点击“一键出表”，在表格列确认窗口选择“默认生成”或“保存并生成”。

结果会自动保存到当天批次总目录里的 `输出结果` 文件夹，不需要手动设置输出目录。

执行时中间流程区会显示进度条、百分比、预计剩余时间和滚动终端日志。OCR 阶段按图片张数推进，解析和 Excel 生成按阶段推进。

已有 `paddle_ocr_raw.json` 时，软件只会在它覆盖当前全部图片、没有失败、没有空文本、没有混入旧图片时跳过整批 OCR；否则会自动重跑 OCR，不会拿失败 OCR 继续生成空白表。
需要调整列、列宽或原图方式时，直接点“一键出表”，在弹出的表格列确认窗口里调整后点“保存并生成”。

默认生成后会同时得到：

- `运输录入_<日期范围>.xlsx`
- `运输录入_<日期范围>_原图` 文件夹
- 文件夹里的 `原图索引.csv`

传给同事时，把 Excel 和 `_原图` 文件夹放在一起发送。Excel 里的 `原图` 列是相对链接，`原图索引.csv` 也记录相对路径，不依赖本机 `C:\Users\...` 绝对地址。

筛选、排序或移动行时，`原图` 链接单元格会跟着该行走。外部图片文件名是在生成 Excel 时固定下来的，不会因为 Excel 里筛选或排序而自动重命名。

## 4. GPS 数据

右上角默认显示 GPS 快照；本机保存过 GPS 授权后，软件启动时会自动刷新一次。点击右上角 GPS 面板或打开“GPS 设置”也会立即刷新。

要让 GPS 真正更新，在“GPS 设置”里点“配置授权”，填入后台接口地址、请求方式、`Authorization` 或 `Cookie`，再点“保存并刷新”。如果开发者工具里显示该接口是 `POST`，还要把“载荷”页里的请求体填到软件的“请求体”框。授权保存在本机配置里，软件会请求 GPS 接口并按后台状态规则统计总车、在线、离线、行驶、停止、报警。

GPS 后台授权不要硬编码进 exe，也不要随便发给外部人员。

## 5. OCR 怎么选

- 普通分享给别人：用“在线智谱 GLM-OCR（默认，发同事推荐）”，一个 exe 即可运行。
- 对方机器有强 NVIDIA GPU，但没装 PaddleOCR / PaddlePaddle GPU：如果仍选择在线智谱，就直接在线跑；如果选择本地 PaddleOCR，软件会先尝试联网补齐本地环境，不会偷偷切回在线 OCR。
- 对方机器有 5090 / 高显存 NVIDIA GPU，并且 PaddleOCR、PaddlePaddle GPU、CUDA、`paddle.utils.run_check()` 都通过：切到“本地 PaddleOCR PP-OCRv5（需安装环境）”。

软件的“OCR 设置”里可以查看推荐、方案库、检查环境、单独运行 OCR、安装本地 OCR 或查看官方资料。检测到 5090 只代表硬件强，不代表本地 OCR 已装好；本地 OCR 还必须找到已安装 PaddleOCR 和 GPU 版 PaddlePaddle 的 `python.exe`。

当前可直接调用的主方案：

- 在线智谱 GLM-OCR：默认方案，适合直接发同事使用。
- 本地 PaddleOCR PP-OCRv5：本地高级方案，适合 5090 这类强显卡配置完整后批量跑。

方案库里的备选方向：

- 通义 Qwen-OCR / Qwen-VL：优先作为下一步在线备选和疑问行复核。
- PaddleOCR PP-StructureV3 / PP-ChatOCRv4：适合表格、版式和关键信息抽取增强。
- 本地 Qwen2.5-VL / GOT-OCR2 / Surya：适合作为疑问行、本地复核和抽样对比，不默认全量跑。
- Mistral OCR：更偏 PDF/文档解析场景，可作为海外或 PDF 备选。

规则：不要默认多引擎全量乱跑。先用主引擎跑整批，再对疑问行用备选方案复核，最后人工确认。

## 6. 本地 PaddleOCR 可选项

本地 PaddleOCR 是高级模式，不是普通分享的前置条件。

选择本地 PaddleOCR 后，软件会优先使用已有 Python；找不到 Python 时会尝试通过 Windows `winget` 安装 Python 3.12，然后安装 PaddleOCR 和 PaddlePaddle。50 系显卡会按 PaddleOCR 官方 50-series wheel 路线处理；GPU 没通过但 CPU Paddle 可用时，也可以用 CPU 本地模式，只是速度会慢。

档位：

- 本地稳妥档（准确优先）：优先兼容，作为本地基线。
- 本地快速档（5090 推荐）：高性能 GPU 优先尝试。
- 本地极限档（TensorRT/FP16）：只在 TensorRT 环境验证通过后使用。

本地极限档不要默认给别人开，必须先小样本确认。

## 7. 下载入口

`发布下载页.html` 是本地下载页模板。把 `发布下载页.html` 和 `做账执行工具.exe` 放到网盘、对象存储或内网 HTTP 目录后，别人点击页面里的下载按钮即可下载 exe。

浏览器不能安全地“下载后自动执行环境安装脚本”。本地 OCR 环境补全已经做进 exe 的“补全本地 OCR”按钮里。

## 8. 常见问题

### 在线 OCR 报 API Key 错误

确认 Key 有效，并且不要重复填写 `Bearer `。软件兼容裸 Key 和 `Bearer Key` 两种写法。

### 点击一键出表后提示 OCR 没跑通

这是保护机制。OCR 返回失败、空文本或网络/API Key 不可用时，软件会停止生成 Excel，避免输出一张看起来有行数但字段全错的表。先按弹窗提示处理 Key、网络或本地 OCR 环境，再重新点“一键出表”。

### 本地 OCR 报 paddleocr=False

说明选中的 Python 没装 PaddleOCR。点击软件里的“补全本地 OCR”，或手动运行：

```powershell
python -m pip install --upgrade paddleocr
```

### CUDA 不可用

`paddle.is_compiled_with_cuda()` 如果是 `False`，说明当前环境不是 GPU 版 PaddlePaddle。

50 系显卡必须按 PaddleOCR / PaddlePaddle 官方当前说明安装对应 GPU wheel。不要随便复制旧机器的 CUDA 包。

### 原图链接打不开

确认 Excel 旁边的 `_原图` 文件夹没有被改名、移动或漏发。

### 格内嵌图模式图片打不开或不在单元格内

必须使用新版 Excel / Excel 365 打开。旧版 Excel 对单元格图片兼容性不足，不能作为降低交付标准的理由。
